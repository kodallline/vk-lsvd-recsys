# VK-LSVD: коллаборативные vs контентные сигналы в рекомендациях

Сравнительное исследование алгоритмов коллаборативной фильтрации и контент-обогащённого ранжирования на датасете коротких видео **[deepvk/VK-LSVD](https://huggingface.co/datasets/deepvk/VK-LSVD)**.

**Сайт проекта:** _включите GitHub Pages (Settings → Pages → ветка `main`, папка `/docs`)._ · **Отчёт:** [`report/VK-LSVD_report.pdf`](report/VK-LSVD_report.pdf) · **Ноутбук:** [`notebook/VK-LSVD_recsys.ipynb`](notebook/VK-LSVD_recsys.ipynb)

## Результат в одной строке
На плотном тёплом каталоге сильнейший метод — **ALS**; контентное слияние прироста **не** даёт (парный бутстрап, p≈0.59). Но на холодных айтемах контент почти **удваивает** HitRate@10 → контент нужен как cold-start / long-tail сигнал, а не как глобальный ре-ранкер.

## Основные метрики (тестовый сплит, полное ранжирование)
| Модель | HitRate@1 | HitRate@10 | NDCG@10 | Coverage@10 |
|---|---|---|---|---|
| ALS (CF only) | 0.0069 | 0.0815 | 0.0092 | 0.0426 |
| ItemKNN | 0.0029 | 0.0747 | 0.0082 | 0.0175 |
| UserKNN | 0.0059 | 0.0766 | 0.0086 | 0.0160 |
| Content only | 0.0010 | 0.0128 | 0.0015 | 0.0151 |
| ALS+Content (a=0.5, d=8) | 0.0079 | 0.0796 | 0.0094 | 0.0430 |

## Cold-start (15% отложенных айтемов, ранжирование внутри холодного пула)
| Модель | HitRate@10 | NDCG@10 |
|---|---|---|
| ALS (CF only) | 0.0171 | 0.0052 |
| Content only | 0.0328 | 0.0103 |
| ALS+Content | 0.0328 | 0.0103 |

## Данные
Полный VK-LSVD ~452 ГБ. Эксперименты на подвыборке `up0.001_ip0.001`, урезанной до 1500 пользователей / 11522 видео / 152660 положительных взаимодействий. Единый контентный эмбеддинг (64-мерный, упорядоченные компоненты) → размерность варьируется как «сила контентного сигнала» вместо раздельных текст/изображение.
